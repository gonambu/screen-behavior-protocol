## デモの目的

UIDPで記述されたUI仕様をAIに渡すと、曖昧さなくコードを生成できることを示す。

### 従来の問題

```
「ToDoアプリを作って」
→ AIは何を作ればいいか分からない
  - 見た目は？
  - 状態管理は？
  - どんな機能？
```

### UIDPを使った場合

```
simple-todo.uidp.yaml を渡す
→ AIは仕様を正確に解釈してコードを生成
  - 構造が明確
  - 状態管理が定義済み
  - アクションが手順化
```

## 使い方

### 1. AIにUIDPファイルを読ませる

```
以下のUIDPファイルを読み、React + MUIで実装してください。

[simple-todo.uidp.yamlの内容]
```

### 2. 生成されるコード

AIは以下を生成する：
- Reactコンポーネント（`TodoList.tsx`）
- 状態管理（`useState` / `useReducer`）
- ローカルストレージ永続化
- MUIコンポーネントの適切な使用

## UIDPの利点

| 観点 | 自然言語 | UIDP |
|------|---------|------|
| 構造 | 曖昧 | 明確（コンポーネントツリー） |
| 状態 | 暗黙的 | 明示的（state, computed） |
| 振る舞い | 解釈が分かれる | 手順化（actions.steps） |
| 再現性 | 低い | 高い |

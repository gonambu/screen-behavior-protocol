## デモの目的

UIDPで記述されたUI仕様をAIに渡すと、**画面遷移を含む複雑なアプリでも**曖昧さなくコードを生成できることを示す。

### 従来の問題

```
「ToDoアプリを作って。一覧と詳細画面がある」
→ AIは何を作ればいいか分からない
  - 画面遷移は？（Router? 条件分岐?）
  - 状態共有は？（Context? Redux? props?）
  - URL設計は？
```

### UIDPを使った場合

```
simple-todo.uidp.yaml を渡す
→ AIは仕様を正確に解釈してコードを生成
  - screens: 画面構成が明確
  - globals: 状態共有が定義済み
  - flows: 画面遷移が明示的
  - route: URL設計が定義済み
```

## UIDPの利点

| 観点 | 自然言語 | UIDP |
|------|---------|------|
| 構造 | 曖昧 | 明確（コンポーネントツリー） |
| 状態 | 暗黙的 | 明示的（state, computed, globals） |
| 画面遷移 | 解釈が分かれる | 明確（flows, navigate） |
| 振る舞い | 解釈が分かれる | 手順化（actions.steps） |
| 再現性 | 低い | 高い |

# UIDPなし実装: シンプルToDo

`comparison.md` の図（画面遷移図、ワイヤーフレーム、機能要件）のみを参照して実装。

## 起動方法

```bash
npm install
npm run dev
```

## 実装時の判断

図を見て自然に実装した結果：

| 項目 | 実装 | 根拠 |
|------|------|------|
| URL設計 | `/todos`, `/todos/:id` | RESTful設計が一般的 |
| ID生成 | `crypto.randomUUID()` | 衝突を避けるため |
| 日時 | ISO8601文字列 | JSON化しやすい |
| 追加位置 | 末尾 | 配列のデフォルト動作 |
| 削除確認 | MUI Dialog | MUIを使っているので |
| 空状態 | 未実装 | 図に無い |
| トースト | 未実装 | 図に無い |

## 図に無くて判断が必要だった点

1. **空の場合の表示** - 図にはタスクが3件ある状態しか無い
2. **タスクの追加位置** - 先頭か末尾か不明
3. **存在しないIDへのアクセス** - エラー画面のデザインが無い
4. **編集時のバリデーション** - 空文字の扱いが不明
5. **削除確認ダイアログのデザイン** - 「確認ダイアログ付き」としか書いていない

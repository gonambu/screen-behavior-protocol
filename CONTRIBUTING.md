# 貢献ガイドライン

UIDPへの貢献に興味を持っていただきありがとうございます。

---

## 貢献の種類

### 1. 仕様へのフィードバック

仕様に関する質問、提案、問題点の報告を歓迎します。

- **質問**: Discussionsで質問してください
- **提案**: Issueで提案してください
- **バグ報告**: Issueで報告してください

### 2. サンプルの追加

様々なUIパターンのサンプルを追加することで、UIDPの実用性を示すことができます。

#### サンプル追加の手順

1. `examples/` ディレクトリに新しいファイルを作成
2. ファイル名は `{カテゴリ}-{名前}.uidp.yaml` 形式
3. ファイルの先頭にコメントで説明を記載
4. PRを作成

#### サンプルの要件

- SPEC.mdに準拠していること
- コメントで設計意図を説明していること
- 実用的なUIパターンであること

### 3. マッピング定義の追加

各フレームワーク向けのマッピング定義を追加することで、UIDPの適用範囲を広げることができます。

#### マッピング追加の手順

1. `mappings/` ディレクトリに新しいファイルを作成
2. ファイル名は `{フレームワーク名}.yaml` 形式
3. PRを作成

#### マッピングの要件

- SPEC.mdで定義されたコンポーネントをカバーしていること
- 対象フレームワークのバージョンを明記すること
- 変換ルールが明確であること

### 4. ツール実装

パーサー、バリデーター、コード生成器などのツール実装を歓迎します。

別リポジトリでの開発を推奨しますが、このリポジトリへのリンク追加は歓迎します。

### 5. ドキュメント

- 翻訳（英語化）
- チュートリアル
- ユースケース説明

---

## 開発環境

### 必要なツール

- Git
- テキストエディタ（VSCode推奨）
- YAMLリンター（任意）

### リポジトリのクローン

```bash
git clone https://github.com/{org}/uidp.git
cd uidp
```

---

## PRの作成

### ブランチ命名規則

- 機能追加: `feature/{機能名}`
- バグ修正: `fix/{問題の説明}`
- ドキュメント: `docs/{内容}`
- サンプル追加: `example/{サンプル名}`

### コミットメッセージ

```
{タイプ}: {簡潔な説明}

{詳細な説明（任意）}
```

タイプ:
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメント
- `example`: サンプル追加
- `refactor`: リファクタリング

### PRテンプレート

```markdown
## 概要

{変更の概要}

## 変更内容

- {変更点1}
- {変更点2}

## 関連Issue

Closes #{Issue番号}

## チェックリスト

- [ ] SPEC.mdに準拠している
- [ ] 既存のサンプルに影響がない
- [ ] ドキュメントを更新した（必要な場合）
```

---

## コードレビュー

PRは以下の観点でレビューされます：

1. **仕様準拠**: SPEC.mdに準拠しているか
2. **可読性**: 人間が読んで理解できるか
3. **一貫性**: 既存のサンプルやドキュメントと一貫しているか
4. **完全性**: 必要な情報が揃っているか
